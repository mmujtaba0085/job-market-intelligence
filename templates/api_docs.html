API documentation

Read-only JSON access to the job data behind this site — the same data the dashboards are built on.

{{ icons.lock(18) }} Authentication

Every request needs an API key. Generate one yourself at /auth/me/keys — no admin approval needed.

Send it in a header, either form works:

X-API-Key: jmi_your_key_here
Authorization: Bearer jmi_your_key_here

API keys are read-only — they can never create, edit, or delete anything, regardless of scope. Each key has its own hourly rate limit (set when you create it) and a fixed set of scopes it's allowed to touch.

{{ icons.briefcase(18) }} GET /api/jobs

The main data feed. Returns job postings sorted by their own posted date (most recent first) — not by when we happened to collect them.

Query parameters

ParamDefaultDescription
limit50Rows to return, max 200 per request.
offset0Pagination offset. Page through by incrementing by limit each request.
market—Filter to one market: pakistan_jobs_all, ai_ml_global, or swe_backend_global.
exclude_market—Inverse of market — everything except the given market.

Example request

curl -H "X-API-Key: jmi_your_key_here" \
  "https://{{ request.host }}/api/jobs?market=pakistan_jobs_all&limit=20"

Example response

{
  "jobs": [
    {
      "job_id": 99636,
      "title": "Senior Backend Engineer",
      "company": "Acme Corp",
      "location": "Lahore, Punjab",
      "country": "Pakistan",
      "remote_type": "on-site",
      "posted_date": "2026-07-06",
      "source_name": "Pakistan Jobs Bank",
      "market_id": "pakistan_jobs_all",
      "url": "https://www.pakistanjobsbank.com/Jobs/..."
    }
  ],
  "limit": 20,
  "offset": 0
}

Paginate by calling again with offset increased by limit each time, until jobs comes back empty.

Note on source_name: the site owner can choose to hide real source names from non-admin accounts. When that's on, you'll see generic labels like "Source A" instead of e.g. "Himalayas" — the job data itself is unaffected either way.
